消息通知之邮件【Notifiable 】

在开始之前，先抛出一个问题，想想利用消息通知来发送邮件和利用模型观察者发送邮件有什么区别？

带着这个问题来做消息通知的邮件消息！😀

首先创建通知：

php artisan make:notification EmailNotifiable
//会生成到app/Notifications/ 文件夹里

不同的是，需要将模型观察者中的直接利用Mail发送邮件代码替换掉，换为用Notification门面（还可以用Notifiable trait 的notify 方法）：

public function created(Email $email)
    {
       // Mail::to($email)->send(new EmailSend($email));//替换为下面的
       //用 facade 发送通知的话，要把可接收通知的实体和通知的实例传递给 send 方法
       Notification::send($email, new EmailNotifiable($email));
    }

指定发送频道：

每个通知类都有个 via ()方法，它决定了通知在哪个频道上发送。开箱即用的通知频道有 mail, database, broadcast, nexmo,和 slack，用的比较多的就是前两种，这里我用的是mail:

//EmailNotifiable.php
public function via($notifiable)
{
    return $notifiable->prefers_sms ? ['nexmo'] : ['mail', 'database'];
}
//因为是邮件通知，数据库通知为toDatabase或toArray
public function toMail($notifiable)
{
    $url = url('/invoice/'.$this->invoice->id);
    return (new MailMessage)
                ->greeting('Hello!')
                ->line('One of your invoices has been paid!')
                ->action('View Invoice', $url)
                ->line('Thank you for using our application!');
}

这里也可以Markdown 邮件通知，和直接发送邮件一样

php artisan make:notification EmailNotifiable --markdown=email.template

与所有其他邮件通知一样，使用 Markdown 模板的通知应在其通知类上定义一个 toMail方法。 但是，不使用 line 和 action 方法来构造通知，而是使用 markdown方法来指定应该使用的 Markdown 模板的名称：

public function toMail($notifiable)
{
    $url = url('/invoice/'.$this->invoice->id);
    return (new MailMessage)
                ->subject('Invoice Paid')
                ->markdown('email.template', ['url' => $url]);
}

这个时候就大概清楚了，利用消息通知来发送邮件和利用模型观察者发送邮件有什么区别？

模型观察者是直接利用邮件门面来发送邮件的，而消息通知，是利用Notification门面来发送一个消息的，只是这个消息走的是邮件通道。然后既然要走邮件通道，就需要相关的文件，所以要创建一个通知，在这个通知类的via()方法中定义通知要走的频道，然后在下面定义要走的频道对应的方法，例如邮件通道的toMail。

所以也可以利用模型观察者来触发这个通知，然后这个通知走的是邮件，相对于直接在观察者中发邮件，多了一步“走哪个通道”选择。
