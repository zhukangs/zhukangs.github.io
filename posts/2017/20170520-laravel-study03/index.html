homestead常用

1.运行 Laravel Mix      npm run watch-poll
2.启动 Homestead        vagrant up   vagrant ssh  exit   vagrant halt
3.vagrant provision 是命令 Vagrant 重新加载 Homestead.yaml 配置；
  vagrant reload 是重启虚拟机使更改生效。

    执行 $ npm run watch-poll 或者 npm run dev（我是执行的后者）

Windows：安装composer

直接下载 composer.phar，地址：https://dl.laravel-china.org/composer.phar
把下载的 composer.phar 放到 PHP 安装目录
新建 composer.bat, 添加如下内容，并保存：
@php "%~dp0composer.phar" %*

使用 Composer 镜像加速有两种选项

- 选项一：全局配置，这样所有项目都能惠及（推荐）；
      composer config -g repo.packagist composer https://packagist.laravel-china.org
- 选项二：单独项目配置；
      composer config repo.packagist composer https://packagist.laravel-china.org
  取消镜像
      composer config -g --unset repos.packagist
  composer 命令后面加上 -vvv （是3个v）可以打印出调错信息
      $ composer -vvv create-project laravel/laravel blog
      $ composer -vvv require psr/log

新增站点

  • 在 Homestead.yaml 文件中新增 larabbs 应用的 sites 和 databases 的相关设置：
---
ip: "10.1.1.33"
memory: 2048
cpus: 1
provider: virtualbox

authorize: ~/.ssh/id_rsa.pub

keys:
    - ~/.ssh/id_rsa

folders:
    - map: D:/Homestead/Code
      to: /home/vagrant/Code

sites:
    - map: local.larabbs.com
      to: /home/vagrant/Code/larabbs/public
    - map: larabbs02.test
      to: /home/vagrant/Code/larabbs02/public

databases:
    - larabbs
    - larabbs02

# blackfire:
#     - id: foo
#       token: bar
#       client-id: foo
#       client-token: bar

# ports:
#     - send: 50000
#       to: 5000
#     - send: 7777
#       to: 777
#       protocol: udp

  • 主要设置了 sites 和 databases 两项。 sites 会将域名 larabbs.test 映射到虚拟机的 /home/vagrant/Code/larabbs/public 文件夹上，而 databases 则为新创建的项目指定数据库名

  • 重启虚拟机

cd ~/Homestead && vagrant provision && vagrant reload

小知识点

  • app()->getLocale() 获取的是 config/app.php 中的 locale 选项 zh-CN

  • @yield('title', 'LaraBBS') 继承此模板的页面，如果没有定制 title 区域的话，就会自动使用第二个参数 LaraBBS 作为标题前缀

<meta name="csrf-token" content="{{ csrf_token() }}">

  • asset('css/app.css') 使用当前请求的协议（ HTTP 或 HTTPS ）为资源文件生成一个 URL，以我们的情况，会生成 http://larabbs.test/css/app.css

  • 辅助函数 需要新建一个空文件 bootstrap/helpers.php ，在 bootstrap/app.php 文件的最顶部进行加载：<?php require_once __DIR__ . '/helpers.php';

<?php
function route_class()
{
    return str_replace('.', '-', Route::currentRouteName());
}
//此方法会将当前请求的路由名称转换为 CSS 类名称，作用是允许我们针对某个页面做页面样式定制
  • 可以在应用程序的任何位置使用全局 config 函数来访问配置值
$value = config('app.timezone');
//运行时设置配置值
config(['app.timezone' => 'America/Chicago']);
  • 使用 env() 方法可知其优先读取 .env 文件中的 APP_NAME 选项
'name' => env('APP_NAME', 'Laravel'),  //config/app.php
